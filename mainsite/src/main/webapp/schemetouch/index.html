SchemeTouch

Motivations

  1. A touchscreen does not discriminate on method of input as strongly as a mouse/keyboard does.
  2. The interface adapts. We like this.
  3. When the syntax is simple, the computer knows best. We hope this will be very useful to beginners, and to those who are extremely tired or drunk.

While text editing is the time-tested default way to write computer programs, very rarely does a computer run code as it is. Whether compiled or interpreted, code goes through all sorts of intermediate stages between how it appears to a human programmer and how the computer actually uses it.

We decided one day to ask why.

We had a lot of thinking and planning to do. We didn't want yet another drag-and-drop editor - that's overdone, and more importantly, slow. Furthermore, we weren't about to spend years developing a new programming language - whatever we built had to be compatible with existing code.

What we came up with was something of an inversion of the 1st intermediate stage code usually goes through - parsing. Instead of turning text into a more abstract representation, we decided that the computer, being all-knowing in such things, should suggest the next course of action from what it knows is available. We save the user from ever making syntax errors, and furthermore relieve the burden of typing on a touchscreen.
